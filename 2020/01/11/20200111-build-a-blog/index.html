How to build a free personal blog using Github + Hexo

Posted on 2020-01-11 Edited on 2020-01-14 In Skill

Introduction

This article will help you to get a free personal blog using Github + Hexo on Windows 10, even you did not know these two tools before.

Basics

GitHub Pages

What is GitHub Pages? - GitHub Help

GitHub Pages is a static site hosting service that takes HTML, CSS, and JavaScript files straight from a repository on GitHub, optionally runs the files through a build process, and publishes a website.

Hexo

What is Hexo? - Hexo Docs

Hexo is a fast, simple & powerful blog framework based on Node.js, which can generates static files with a beautiful theme from your writen posts in Markdown (or other markup languages) in seconds.

Github + Hexo

After setting up the environment, you can write articles on your personal computer in Markdown, then generate html websites and post on your blog.

  • Pros: It is totally free, static and fast. You can modify the pages, including theme and content, as you like. It wll be stored on Github safely. It is easy to transfer.
  • Cons: It is dependent on local environment and mainly suits individual blogs. You need to generate web pages every time you make changes. Access to Github is needed.

Environment setting

Install Node.js and Git

  1. Download the packages:
  • Node.js https://nodejs.org/en/
  • Git https://git-scm.com/downloads

  1. Continue to click “Next” button to select the default settings when installing the two packages.

  2. After the installation, press “Win + R”, input “cmd” to open command window, input node -v, npm -v, and git --version separately. If versions are given, the environment has been successfully setted.

Now you can use Git Bash function to substitute cmd when you need to run command lines under certain folder path (just right click and select “Git Bash” in the folder).

Install Hexo

  1. Before install Hexo, you are advised to create a new folder to store all the files that will be created for the blog and name it, such as “Blog”.

  2. Right click to lauch Git Bash under the folder “Blog”. Input npm install -g hexo-cli to install Hexo.

  3. After the installation, input hexo -v to see if the installation is successful. If the version is given, it is successfully installed.

From now on, all the commond lines for the blog are supposed to run under the “Blog” folder or its child folders.

Initiate local webpage

  1. Open Git Bash, input and run hexo init to initiate the “Blog” folder. Then input and run npm install to install components.
  2. Input and run hexo g to generate a static webpage.
  3. Input and run hexo s , then you can get a link http://localhost:4000/. Paste the link into your Web Browser, then you can see your local blog page in a default Hexo style. You can close the local server by press “Ctrl + C”.

Now you get a local blog on your personal computer, but it has not been online yet. Under the “Blog” folder, now you can find several newly-created files:

  • node_modules: dependent package
  • public：to store the generated pages;
  • scaffolds：to store module files;
  • source：to store your articles and resources like images and files;
  • themes：theme files
  • _config.yml: site configuration file
  • package.json: application data

Get your page online

Create a Github repository for your blog

  1. Register a Github account.
  2. Click into “Your repositories” and create a new repository by clicking the “New” button. Name the repository as “your github username.github.io”.
  3. This will be your blog site: “your github username.github.io”.

Set up SSH

By setting up SSH, you will be able to connect your local webpage with your github account via git.

  1. Open Git Bash, input and run git config --global user.name "your GitHub username", and git config --global user.email "your GitHub email".
  2. Input and run ssh-keygen -t rsa -C "your GitHub email" to create SSH key.
  3. Enter into the folder [C:\Users\用户名.ssh], open the “id_rsa.pub” file using Notebook. Use “Ctrl + A” and “Ctrl + C” to select and copy all the content in the file.
  4. Log in your github account, and enter into the setting page. Click into the “SSH and GPG keys” page, Click “New SSH key” button, paste the content you copied from “id_rsa.pub” file. Click the “Add SSH key” and you can see a SSH key is created.
  5. Go back to the Git Bash, input and run ssh -T git@github.com. If it asks “Are you sure you want to continue connecting?”, input yes and press “Enter”. The connection is successful it says “Hi username! You’ve successfully ```”.

  3. Now you can call the deploy command:

  5. If you want to change the title or other basic settings of the blog, open the “_config.yml” file under the “Blog” folder and then you can change the title, author name and other things.

Create and publish new articles

All the articles will be stored under the folder “Blog\source_posts” in Markdown.

  1. Open Git Bash under the “Blog” folder, install an extension by running npm i hexo-deployer-git;
  2. Input and run hexo new "title", then you can find a newly created Markdown file under “Blog\source_posts” folder named “title.md”.
  3. Open “title.md” with a Markdown ediing software and write your article here.
  4. Publish the new article by running hexo g and hexo d. Every time you create a new article, you need to run these two lines to publish your new contents.

Basic markdown syntax: https://help.github.com/en/github/writing-on-github/basic-writing-and-formatting-syntax

Theme settings

Change the theme

You probably do not like the default Hexo page style, of which the theme is “landscape”. There are acctually more choices and you can find them here. Now you should know how to change your blog theme into the one you like.

  1. Find the theme you like in the website and click into its github pag. Here we take the “Next” theme as an example, whose github page is https://github.com/theme-next/hexo-theme-next.
  2. Clone the theme to your “themes” folder by running
  3. Open the “_config.yml” file under the “Blog” folder, search the keyword “theme” and change it from “theme: landscape” into “theme: Next”.

Custome settings

Change the domain name

If you do not like “github.io” suffix of your blog and want to change it to a personal domain name, you need to buy one first from namesilo, Tecent or other suppliers.

  1. Open the “Blog\source” folder, create a file named “CNAME” without suffix, open it wiht Notebook and write your own domain name into it and save.
  2. Run hexo g and hexo d to deploy it to your blog.
  3. Now you can visit your blog with your own domain name.

Article display in the lists

The default display mode is display the whole texts, which reduces it readability when the articles are long. In this case, you can add a <!--more--> in appropriate location in the article. By doing this, only the contents before this code will be displayed as an abstract and the remaining will be hide with a “Read more” button.

Change article templates

You can change the template by modifying the file “Blog/scaffolds/post.md”
